# Agent guide

Pointers for AI agents (Claude Code, etc.) working in this repo. Humans should read the
README first.

## Repo at a glance

- macOS SwiftPM app (`Package.swift`, `Sources/SuperMD/`), built into a `.app` bundle
  by `scripts/build-app.sh`.
- No Xcode project — everything is driven from the command line.
- Releases are produced **locally** (signing cert + Sparkle key live in the developer's
  keychain, not in CI). See [docs/release.md](docs/release.md).

## Release work — read this before touching versions or tags

A release is **four artifacts in lockstep**: a `Release vX.Y.Z` commit (bumping
`Resources/Info.plist` + `Sources/SuperMD/SuperMDApp.swift`), a regenerated `appcast.xml`,
a pushed git tag, and a GitHub release with **both** the `.dmg` and `.zip` attached.
Pushing only the tag — or running `gh release create` without the artifacts — leaves
auto-update broken for every installed copy.

Always run `./scripts/release.sh` end-to-end; it builds, notarizes (twice), signs, and
regenerates the appcast. The full checklist and recovery procedure for a half-finished
release are in [docs/release.md](docs/release.md).

## Conventions

- Commit message style: short imperative ("Add X", "Fix Y", "Release vX.Y.Z"). Recent
  history under `git log` is the source of truth.
- Don't introduce CI-side signing or move secrets into GitHub — the project's threat model
  keeps the Developer ID cert and Sparkle private key on the maintainer's Mac only.
- `appcast.xml` is generated by `scripts/release.sh`. Don't hand-edit it.
